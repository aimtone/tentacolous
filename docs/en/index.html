Spring Boot + PostgreSQL

Tentacolous

React to real database changes with annotated Java methods.

Tentacolous detects INSERT, UPDATE and DELETE from PostgreSQL, even when the change comes from another API, a script, a SQL console or a legacy system. No JPA events, no hidden magic: triggers, an event table and a Spring poller.

@UponInserting(entity = Persona.class)
public void onPersonaInserted(Persona persona) {
    notify(persona.getEmail());
}
Essentials

A simple bridge between PostgreSQL and your Java code.

This page keeps the story focused: what Tentacolous solves, how it is used and when to open the full documentation.

DB

Changes outside your app

Detect changes made by other systems, scripts or SQL tools, not only writes made by your Spring service.

JV

Annotated listeners

Write methods with @UponInserting, @UponUpdating or @UponDeleting and receive the Java entity.

PG

Clear infrastructure

Use PostgreSQL triggers, db_change_event, processing states and configurable retries.

Why it exists

Because the database also speaks.

In many systems, the database receives changes from more than one place. Tentacolous turns those changes into extension points inside Spring Boot, without pretending to be Debezium or native CDC.

01

The source is PostgreSQL. The event starts when the table changes.

02

The reaction lives in Java. The listener receives a deserialized domain object.

03

Production with control. You can create, validate or disable automatic schema management.

How it works

A short, observable flow.

Tentacolous registers listeners, creates or validates PostgreSQL infrastructure, captures changes and dispatches events to your methods.

PostgreSQL tableINSERT / UPDATE / DELETE
->
db_change_eventpayload JSON + status
->
Spring pollerbatch + retries
->
Java listener@UponInserting, @UponUpdating, @UponDeleting
Quick start

Install, configure, annotate.

The current local integration uses Maven local and the artifact com.aimtone.tentacolous:tentacolous:0.0.1-SNAPSHOT.

1. Install the librarybash
mvn clean install
2. Add the dependencyxml
<dependency>
  <groupId>com.aimtone.tentacolous</groupId>
  <artifactId>tentacolous</artifactId>
  <version>0.0.1-SNAPSHOT</version>
</dependency>
3. Configure Tentacolousyaml
tentacolous:
  enabled: true
  schema-management: auto
  event-table: db_change_event
  poll-interval: 1s
4. Listen to a tablejava
@UponUpdating(entity = Payment.class)
public void onPaymentUpdated(Payment payment) {
    sync(payment);
}
