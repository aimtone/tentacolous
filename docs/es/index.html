Tentacolous

Reacciona a cambios reales de base de datos con metodos Java anotados.

Tentacolous detecta INSERT, UPDATE y DELETE desde PostgreSQL, aunque el cambio venga de otra API, un script, una consola SQL o un sistema legacy. Sin eventos JPA ni magia escondida: triggers, tabla de eventos y un poller Spring.

public void onPersonaInserted(Persona persona) {

Por que existe

Porque la base tambien habla.

En muchos sistemas, la base de datos recibe cambios desde mas de un lugar. Tentacolous convierte esos cambios en puntos de extension dentro de Spring Boot, sin pretender ser Debezium ni CDC nativo.

01

La fuente es PostgreSQL. El evento nace cuando la tabla cambia.

02

La reaccion vive en Java. El listener recibe un objeto de dominio deserializado.

03

Produccion con control. Puedes crear, validar o desactivar la gestion automatica de schema.

Como funciona

Un flujo corto y observable.

Tentacolous registra listeners, crea o valida infraestructura PostgreSQL, captura cambios y despacha eventos hacia tus metodos.

Inicio rapido

Instala, configura, anota.

La integracion local actual usa Maven local y el artefacto com.aimtone.tentacolous:tentacolous:0.0.1-SNAPSHOT.

1. Instala la libreriaterminal
mvn clean install
2. Agrega la dependenciaxml
<dependency>
  <groupId>com.aimtone.tentacolous</groupId>
  <artifactId>tentacolous</artifactId>
  <version>0.0.1-SNAPSHOT</version>
</dependency>
3. Configura Tentacolousyaml
tentacolous:
  enabled: true
  schema-management: auto
  event-table: db_change_event
  poll-interval: 1s
4. Escucha una tablajava
@UponUpdating(entity = Payment.class)
public void onPaymentUpdated(Payment payment) {
    sync(payment);
}
