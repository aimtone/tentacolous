# Contributing to Tentacolous

Thank you for your interest in contributing to Tentacolous.

Tentacolous is developed and maintained by the official project maintainers.
Contributions are welcome and may help improve the project, but all changes
must go through the official repository.

## How to Contribute

1. Check existing issues before proposing a change.
2. Open an issue describing the problem or proposed feature.
3. Wait for discussion and approval before implementing major changes.
4. Submit a Pull Request with a clear description of your changes.

## Pull Requests

Pull Requests should:

- Follow the existing code style.
- Include tests when applicable.
- Include documentation updates when changing public APIs.
- Explain the reason and impact of the change.

All Pull Requests are reviewed by the Tentacolous maintainers.

## Acceptance of Contributions

Submitting a contribution does not guarantee that it will be accepted.

The Tentacolous maintainers reserve the right to:

- Accept or reject contributions.
- Modify submitted code before inclusion.
- Decide the roadmap and direction of the project.

## Intellectual Property

By submitting a contribution, you agree that accepted changes may be incorporated
into the official Tentacolous project and distributed under the Tentacolous
license.

You retain ownership of your original contribution, unless otherwise agreed.

## Development Rules

Please do not:

- Submit unrelated changes.
- Introduce breaking changes without discussion.
- Publish modified versions of Tentacolous outside the official project.

Thank you for helping improve Tentacolous.
